# 📖 KIS-Vibe-Trader 사용자 매뉴얼 (EXE 버전)

본 매뉴얼은 `KIS-Vibe-Trader.exe` 실행 파일을 사용하는 일반 사용자를 위한 가이드입니다. Python 설치 없이 실행 가능하며, TUI(터미널 UI)를 통해 모든 기능을 제어할 수 있습니다.

---

## 1. 시작하기 (Setup)

### A. 준비물
1.  **한국투자증권(KIS) API 키**: [한국투자증권 개발자 센터](https://apiportal.koreainvestment.com/)에서 앱키와 앱시크릿을 발급받으세요. (실전/모의계좌 모두 가능)
2.  **Google Gemini API 키**: [Google AI Studio](https://aistudio.google.com/)에서 발급받으세요. (AI 분석 기능을 위해 필수)
3.  **Groq API 키 (선택 사항)**: [Groq Console](https://console.groq.com/)에서 발급받으세요. Gemini 장애 시 백업 모델로 사용하거나 주 모델로 설정 가능합니다.
4.  **D2Coding 폰트 (필수 권장)**: 완벽한 TUI(터미널) 화면 출력을 위해 배포 파일에 동봉된 `D2Coding.ttf` 폰트를 시스템에 설치해 주세요.
    *   **[Windows]**: 다운로드한 `D2Coding.ttf` 파일을 더블클릭 후 **[설치]** 버튼을 누르거나, 우클릭하여 **[모든 사용자용으로 설치]**를 클릭합니다. 설치 후 현재 사용하는 터미널(명령 프롬프트, PowerShell, Windows Terminal 등) 설정에 들어가 글꼴(Font)을 `D2Coding`으로 변경합니다.
    *   **[Linux]**: 배포된 폰트를 시스템 폴더로 복사한 뒤 캐시를 갱신합니다.
        ```bash
        sudo mkdir -p /usr/share/fonts/truetype/d2coding
        sudo cp D2Coding.ttf /usr/share/fonts/truetype/d2coding/
        fc-cache -fv
        ```
        이후 터미널 에뮬레이터의 설정 창에서 텍스트 글꼴을 `D2Coding`으로 지정합니다.

### B. 첫 실행 및 환경 설정
1.  `KIS-Vibe-Trader.exe`를 더블 클릭하여 실행합니다.
2.  최초 실행 시 키보드의
*   **`K`**: 🧪 모의거래 자가 진단 (드라이런/시간워프) ※ 모의투자 전용
*   **`S`**: ⚙️ 시스템 설정 (Config/Setup)
 Mode]**로 진입합니다.
3.  화면의 안내에 따라 다음 정보를 입력합니다:
    *   `App Key / Secret`: 한국투자증권에서 발급받은 키.
    *   `Account Number (CANO)`: 계좌번호 8자리.
    *   `Is Virtual? (y/n)`: 투자 모드 선택.
    *   `Gemini / Groq API Key`: 사용할 AI 서비스의 키를 입력합니다 (없으면 엔터로 스킵 가능).
    *   **[신규] 텔레그램 알림 설정**: 실시간 매매 및 리포트 알림을 받을 `TELEGRAM_TOKEN`과 `TELEGRAM_CHAT_ID`를 입력합니다. (상세 방법은 3장 참조)
    *   **[신규] 멀티 LLM 설정**: 사용하는 AI 서비스별로 주 모델(Primary)과 부 모델(Secondary)을 지정할 수 있으며, 장애 시 자동으로 다음 모델로 Fail-over 됩니다.
    *   **[신규] AI 자율 매수/매도 모드**: AI 추천 종목을 자동으로 살지(Buy), 그리고 보유 종목에 대해 AI가 매도 결정을 내렸을 때 자동으로 팔지(Sell)를 각각 독립적으로 설정할 수 있습니다.
    *   **[신규] AI 디버그 모드**: 장외 시간에도 테스트를 위해 AI 자동 기능을 강제 활성화할 수 있습니다 (Y/N).
    *   `총 누적 입금액 (Seed)`: 누적 자산 수익을 측정하기 위한 초기 입금 총액.
    *   **[신규] 최대 보유 종목 수 (`MAX_STOCK_COUNT`)**: 동시에 보유할 최대 종목 수를 설정합니다 (1~8). **`Y`** 입력 시 AI가 현재 예수금 규모에 맞춰 3/5/8개 중 최적의 사이즈를 자동 선택합니다.
4.  설정이 완료되면 프로그램이 자동으로 재기동되며 실시간 데이터를 수집하기 시작합니다.

### C. 멀티 LLM (Fail-over) 및 Groq 설정
본 시스템은 안정적인 AI 분석을 위해 여러 대규모 언어 모델(LLM)을 교차 사용하는 **Fail-over** 기능을 지원합니다.

1.  **Groq 활용**: Groq은 매우 빠른 응답 속도를 제공하는 LLM 서비스입니다. Gemini API에 장애가 발생하거나 호출 제한에 걸렸을 때 Groq의 모델(Llama 3 등)이 즉시 분석 업무를 승계합니다.
2.  **우선순위 설정 (Fail-over Sequence)**:
    *   `S:셋업` 메뉴에서 Gemini와 Groq의 키를 모두 입력하면, 분석 시 사용할 모델의 순서를 지정할 수 있습니다 (예: `1:Gemini`, `2:Groq`).
    *   첫 번째 모델이 실패하면 시스템은 지체 없이 두 번째 모델을 호출하여 분석 리포트 생성을 완료합니다.
3.  **비용 절감**: Groq의 무료 티어 모델들을 적절히 섞어 사용하면, Gemini의 유료 토큰 소모를 최소화하면서도 중단 없는 자율 매매가 가능합니다.

### 데이터 동기화 및 안정성 강화 [v1.6.0]
*   **고정 임계치 정책 (Fixed Target)**: AI가 특정 전략(Preset)을 할당했거나 사용자가 수동으로 TP/SL을 조정한 종목은 더 이상 시장 상황(Vibe)에 따른 추가 보정을 하지 않습니다. 이는 화면에 표시된 숫자가 실제 매수/매도 기준과 100% 일치하도록 보장하기 위함입니다.
*   **AI 호출 복원력 (Resilience)**: AI 자율 매매 컨펌 중 일시적인 API 장애가 발생할 경우 최대 3회까지 자동 재시도하며, 단순 호출 실패 시 해당 종목을 '거절(Rejected)' 목록에 넣지 않고 다음 사이클에 즉시 다시 판단하도록 개선되었습니다.
*   **저점 매수 필터 확대**: AI 추천 종목 진입을 위한 하한 등락률 필터를 기존 -1.5%에서 **-8.0%**로 대폭 완화하여 낙폭 과대 우량주의 선취매 기회를 확보했습니다.
    *   `KR:OPEN | US:CLOSED`: 한국 및 미국 시장의 실시간 개장 상태.
    *   `분석: ...`: 실시간 시황 분석 상태 (초기화 중, 분석 중, 분석 완료).
    *   `작업(N): ...`: 현재 시스템이 수행 중인 상세 액션(전략분석, 매매시도, 익절실행 등)과 백그라운드 **스레드 개수(N)**를 표시합니다. 
    *   **[개선]** 매수/매도 성공 등 사용자 동작에 대한 피드백(Status)이 발생하면 이곳에 **노란색으로 최우선 강조**되어 표시됩니다. 화면이 좁을 경우 가독성을 높이기 위해 하단 별도 STATUS 라인은 제거되었습니다.
    *   `시간`: `(스레드수) 요일 HH:MM:SS` 형식의 한국어 시계. (우측 끝에 고정되어 레이아웃 안정성 강화)
    *   **로그 상태**: 로그 정리 작업 시 `로그 정리 중` 알림 표시.
    *   **[신규] 자동 업데이트**: GitHub 최신 릴리스가 감지되면 버전 정보 옆에 **`[NEW v1.x.x]`** 알림이 노란색으로 강조됩니다. (버전은 `Major.Minor.Patch` 단계까지만 비교하여 불필요한 빌드 업데이트 알림 최소화)
    *   **[신규] API 비용 추적**: Gemini API 월간 누적 사용 예상 비용을 실시간으로 추산하여 상단에 표시합니다.

### 로그 관리 시스템 (Log Cleanup)
*   **자동 최적화**: 프로그램 실행 중 1시간마다 백그라운드에서 `error.log`, `trading.log`, `telegram.log`, `trading_logs.json` 파일을 점검합니다.
*   **영업일 기준 관리**: 주말을 제외한 **최근 2일간의 영업일 로그**만 남기고 오래된 데이터는 자동 삭제하여 저장 공간을 절약하고 시스템 성능을 유지합니다.
*   **상태 알림**: 작업 중에는 상단 헤더에 `로그 정리 중` 메시지가 표시되며, 완료 후 트레이딩 로그 창에 결과가 기록됩니다.
2.  **Market Indices**: KOSPI, KOSDAQ, NASDAQ, 환율 등 주요 지수 실시간 표시.
    *   **C Market**: 비트코인 국내가/해외가 및 **김치 프리미엄(PREM)** 표시.
3.  **VIBE Status**: 현재 시장 분위기 (BULL/BEAR/NEUTRAL/DEFENSIVE) 및 AI 교정 메시지.
    *   **[신규] DEMA 추세**: VIBE 옆에 `[DEMA: KSP↑ KDQ↓]` 형태로 지수의 이중 지수 이동 평균(DEMA) 대비 현재 위치를 표시합니다 (↑:상회, ↓:하회). 지수가 DEMA 하단에 위치하고 당일 등락률이 마이너스인 경우 **[강력한 Bear]**로 판정하여 리스크 관리를 강화합니다.
4.  **COMMANDS**: 사용 가능한 모든 단축키 리스트.
5.  **전략 정보 패널**:
    *   `STRAT`: 현재 기본 익절/손절 기준 및 Vibe 보정 적용 후 실시간 값.
    *   `BEAR`: 물타기(하락 방어) 전략 설정 — 트리거(%), 회당 금액, 종목한도, 자동 여부. (🔵 청색 계열)
    *   `BULL`: 불타기(수익 확대) 전략 설정 — 트리거(%), 회당 금액, 종목한도, 자동 여부. (🔴 적색 계열)
    *   **ALGO**: AI 추천매매 전략 설정 — 매수 금액, 종목한도, 누적 집행액. 
    *   **상태 표시기**: 하단 `✨ AI 추천` 헤더에서 **[매수:ON|매도:OFF]**와 같이 자동화 기능의 활성화 여부를 실시간으로 확인할 수 있습니다.
    *   **누적 집행**: 각 전략 라인 끝에 오늘 하루 동안 실제로 매수 집행된 **총 누적 금액**이 실시간으로 표시됩니다.
    *   각 라인 앞에 `*` 표시가 있으면 초기 설정값에서 변경(AI 또는 수동)되었음을 의미합니다.
6.  **Portfolio List**: 보유 종목의 수익률, 평가금액, 실시간 손익 및 동적 TP/SL(익절/손절선). 
    *   **TP/SL 색상**: 익절(TP)은 빨간색, 손절(SL)은 파란색으로 표시되어 가독성을 높였습니다.
    *   프리셋 전략이 할당된 종목은 TP/SL 우측에 `[전략명]` 태그가 시안색으로 표시됩니다.
9.  **[신규] 수급 시그널 (Supply Indicators)**:
    *   종목명 우측에 실시간 수급 상태가 표시됩니다.
    *   `[F↑]`: 외인 매수 우위 | `[P↑]`: 연기금/투신 매집 | `[FP↑]`: 외인/기관 쌍끌이 매수.
    *   이 시그널은 네이버와 KIS 데이터를 실시간 교차 검증하여 도출됩니다.
7.  **AI Recommendations**: AI가 분석한 테마별 주도주 및 ETF 추천 리스트.
    *   **[신규] 진입 가시성 확보**: 종목명 좌측에 `[자동]` 또는 `[수동]` 태그를 표시합니다. 시스템의 자동 매수 허용 범위(당일 등락률 ±8.0% 이내)를 충족하는 종목은 초록색 `[자동]`으로 표시되며, 이를 벗어난 급등/급락주는 회색 `[수동]`으로 표시되어 추격 매수 리스크를 시각적으로 경고합니다.
8.  **Naver Rankings**: 네이버 금융 기반 실시간 인기/거래량 상위 종목.
    *   **Yesterday's Performance**: 하단에 어제 추천된 종목들의 현재 수익률 트래킹 표시. **[신규]** 수익률 높은 순서대로 정렬되며, 종목명과 변동률 색상이 최적화되었습니다.

| **`H`** | **인기 테마 리포트** | 실시간 인기 검색 TOP 10 종목의 테마 분석 및 AI 트렌드 진단 리포트를 확인합니다. |
| **`L`** | **트레이딩 로그** | **[개선]** 최근 체결 내역(TRADE) 및 설정 변경 이력(CONFIG)을 **최신순**으로 조회합니다. 화면 높이 초과 시 오래된 로그는 자동 생략(Omit)됩니다. |
| **`S`** | **환경 설정** | API 키, 계좌 정보, 전략 파라미터를 다시 설정합니다. 최대 보유 종목 수(MAX_STOCK_COUNT)를 변경할 수 있습니다. |
| **`P`** | **성과 리포트** | **[신규]** 명예의 전당(TOP 10 수익/손실), 모델별(G3.1P, G3.1FL, 수동 등) 상세 승률, AI 리밸런싱 제안을 한눈에 확인합니다. |
| **`Q`** | **종료** | **[신규]** "시스템을 종료합니다" 메시지를 1초간 표시한 뒤 프로그램을 안전하게 종료합니다. |
| **`9`** | **프리셋 전략** | 보유 종목에 KIS 10대 공식 매매 전략을 할당합니다. 엔터 시 AI 자동 추천. |
| **`U`** | **자동 업데이트** | **[신규]** GitHub 최신 릴리스를 다운로드하고 프로그램을 즉시 자동 재기동합니다. |

## 4. 핵심 시스템 가이드

### AI 분석 자동화 및 시황 갱신
*   **시작 분석**: 프로그램 실행 시 메인 화면 로딩과 동시에 백그라운드에서 즉시 `8:시황분석`을 수행합니다. 분석이 완료될 때까지 매매 로직은 대기 상태를 유지하여 어제의 낡은 전략으로 매매되는 것을 방지합니다.
*   **주기적 갱신**: 실전 계좌는 **20분**, 모의 계좌는 **60분**마다 자동으로 시황을 재분석하여 최적의 TP/SL 수치를 시스템에 반영합니다.
*   **[신규] 통합 배치 리뷰 (Batch Review)**: 개별 종목별 AI 호출 방식을 폐지하고, 한 번의 API 호출로 전 종목을 동시에 입체 분석하여 속도 및 토큰 소모를 극대화했습니다.
*   **실패 대응**: AI 분석 중 네트워크 오류 발생 시 1회 자동 재시도하며, 지속 실패 시 기존 알고리즘 모드로 안전하게 전환되어 매매가 멈추지 않도록 설계되었습니다.
*   **[신규] 데이터 무결성 보호**: 0원 데이터 수신이나 변수 초기화 오류(`UnboundLocalError`)와 같은 런타임 예외를 방지하기 위해, 데이터 수집 및 지표 산정 전 구간에 걸쳐 강력한 초기화 및 예외 핸들링(`Safe-Check`) 로직이 적용되었습니다.
*   **[신규] AI 토큰 절약 (운영 시간 제한)**:
    *   AI API 호출이 수반되는 자동화 기능은 **08:40 ~ 15:50** 사이에만 작동합니다.
    *   장이 닫힌 시간에는 자동 분석이 중단되어 불필요한 토큰 소모를 방지합니다. 
    *   수동 분석(8, 9번 등)은 언제든 사용 가능합니다.

### AI 자율 매매 (ALGO) 고도화
*   **다중 종목 순차 매수**: AI 추천 리스트 전체를 순회하며, 투자 한도에 여유가 있는 종목들을 순차적으로 매수합니다.
*   **투자 한도 체크**: `ALGO` 설정의 '한도' 금액을 참조하여, (현재 평가액 + 매수 예정액)이 한도를 초과할 경우 해당 종목은 스킵하고 다음 순위 종목으로 넘어갑니다.
*   **최종 구매 컨펌 (Confirm)**: **[핵심]** 매수 버튼을 누르기 직전, Gemini AI에게 현재 뉴스/장세를 바탕으로 "지금 사도 좋은가?"를 한 번 더 묻습니다.
    *   **모델 로깅**: 실제 판단을 내린 모델명(G3.1FL 등)이 로그에 함께 기록됩니다.
    *   AI가 거절(No)할 경우 해당 사유를 트레이딩 로그에 기록하고 당일 매수 대상에서 제외합니다. (⚠️ 거절 사유 예: `[G3.1P]` "최근 악재 뉴스로 인해 단기 변동성 우려")
    *   승인(Yes)된 경우에만 실제 주문이 집행됩니다.
    *   이 과정을 통해 설정된 한도(`MAX_STOCK_COUNT`) 내에서 항상 시장에서 가장 유망한 종목들로 포트폴리오를 유지(Rotation)할 수 있습니다.
*   **AI 자율 매도 (Proactive Exit)**: **[신규]**
    *   **자동 처분**: AI가 시황/뉴스 악재로 'SELL' 판정을 내리면 즉시 시장가 매도를 집행합니다 (자율 매도 AUTO ON 시). 단, 매수 후 1시간 내의 종목은 손실 방지를 위해 AI가 매도를 권고해도 자동으로 차단됩니다(`🛡️ 매도 보호`).
    *   **장외 대응**: 장이 닫힌 시간에 매도 권고가 뜨면 익절/손절선을 현재가 근처(±0.1%)로 타이트하게 조여(Tightening), 익일 장 개시 즉시 매도되도록 준비합니다.
*   **당일 등락률 진입 필터**: **[중요]** 이미 과도하게 급등한 종목의 추격 매수를 방지하기 위해, 당일 등락률이 **-8.0% ~ +8.0%** 범위 내에 있는 종목만 신규 진입을 허용합니다. 이 범위를 벗어난 종목은 화면에 `[수동]`으로 표시되어 자동 매수가 차단됩니다. 특히 상승장(Bull)에서는 등락률이 +4.0%를 넘어가면 상투(고점) 잡기를 방지하기 위해 모멘텀 가점을 점진적으로 삭감합니다.
*   **이동평균선(MA) 입체 분석**: **[신규]** 단순 가격 데이터가 아닌 이중 타임프레임(일봉+분봉) MA를 분석하여 매매 확률을 높입니다.
    *   **일봉(Trend)**: 20일 이동평균선 위(상승추세)인지 아래(하락추세)인지 판단하여 중기 방향성을 확인합니다. 일봉 하락추세 종목은 매수 점수가 자동으로 감축(20%)됩니다.
    *   **분봉(Timing)**: 현재가가 분봉 20MA에 근접(`BUY_ZONE`)했는지 또는 과도하게 이탈(`OVERBOUGHT`)했는지 분석하여 최적의 진입/청산 시점을 포착합니다.
    *   **매수/매도 근거 활용**: 분봉 20MA 지지선 근접(`BUY_ZONE`) 시 AI 승인 확률을 높이기 위한 가점이 부여되며, 반대로 단기 과열(`OVERBOUGHT`, +3% 이상 이격) 시에는 장세와 무관하게 추격 매수를 기계적으로 원천 차단합니다.
    *   **[신규] 지수 DEMA 연동**: 시장 전체 장세 판단 시 지수와 DEMA(20)의 이격도를 AI가 분석하여 "추세 지지/저항" 관점의 고도화된 시황 브리핑을 제공합니다.
    *   **[신규] MA20 데이터 3중 폴백**: 분봉 데이터 소스를 (1) KIS API → (2) 네이버 F-Chart XML 순서로 자동 전환하여 데이터 취득률을 높입니다. 두 소스 모두 실패한 경우 `UNKNOWN` 시그널이 반환되며, 해당 종목은 `score -30pt` 패널티 적용 후 최소 기준 미달 시 자동 매수가 차단됩니다.
    *   **[신규] OVERBOUGHT 하드 차단**: AI의 '공격적 페르소나'가 과열 종목 매수 시도를 우회하는 것을 막기 위해, `OVERBOUGHT` 시그널 감지 시 Gemini 호출 이전에 Python 로직에서 직접 매수를 차단합니다.
    *   **[신규] 수급 사이클 필터링 (Supply Cycle)**: 단순히 당일 수급뿐만 아니라 최근 10일간의 매집 상태를 분석하여 '매매 사이클'이 상승 구간인 종목에 가점을 부여하고, 반대로 수급 이탈이 심각한 종목은 진입을 차단합니다.

*   **[신규] 핑퐁 매매 방지 (Anti-Pingpong)**: 매도 사유별로 같은 종목의 재진입 쿨다운 시간이 다르게 적용됩니다.
    *   **손절/긴급손절**: 24시간 — 지지선이 붕괴된 종목으로 당일 및 익일까지 재진입 금지.
    *   **P4 장마감 손절 / AI 자율 매도**: 8시간 — 오버나이트 리스크 또는 AI 악재 판단에 의한 청산.
    *   **교체 매도**: 4시간 — 다른 종목이 상대적으로 우세했을 뿐 절대 문제가 있는 종목은 아니므로 단기 차단 후 재평가.
    *   이 설정은 `trading_state.json`에 저장되어 프로그램 재시작 후에도 유지됩니다.
*   **[신규] 교체 품질 게이트**: 보유 종목 교체 시 3단계 사전 검증을 통해 불필요한 교체를 방지합니다.
    *   **① OVERBOUGHT 차단**: 후보 종목이 과열 구간(`OVERBOUGHT`)이거나 MA 지표가 없으면 교체 금지.
    *   **② 점수 우위 검증**: 후보 종목의 퀀트 점수가 기존 보유 최저 점수보다 **+15점 이상** 높아야만 교체 허용.
    *   **③ 30분 보유 보호**: 보유 종목 중 취득 후 30분이 지나지 않은 종목이 있으면 교체 대기.

### 전략 자동 재수립 (Time-Stop)
*   종목별로 설정된 **전략 유효 시간**이 만료되면, 시스템이 자동으로 AI에게 해당 종목의 재분석을 요청합니다.
*   분석 결과에 따라 최신 뉴스/차트 흐름을 반영한 **새로운 프리셋 전략과 TP/SL**이 즉시 할당됩니다.
*   만약 AI 분석이 실패할 경우, 기존의 '수익 보존 로직(수익권일 때 TP 하향)'이 Fallback으로 작동하여 안전한 엑싯을 돕습니다.

### 자산 현황 (ASSETS) 해석
*   `S (셋업)` 메뉴에서 **총 누적 입금액(Seed)**을 설정한 경우:
    *   **입금액**: 사용자가 설정한 원금입니다.
    *   **누적수익**: `(총 평가액 - 입금액)`을 역산하여 전체 누적 수익금과 수익률(%)을 정확히 표시합니다.
*   **Total**: (보유 주식 평가액 + 예수금) 합계로, 실제 내 계좌의 총 자산 가치입니다.
*   **Stock**: 현재 보유 중인 모든 주식의 평가 금액 합계입니다.
*   **Cash**: 즉시 매수에 사용 가능한 **예수금(D+2 정산 기준)**입니다.
*   **PnL**: 총 매입 원금 대비 실시간 평가 손익 금액 및 수익률입니다. (시드머니 미설정 시)
*   **금일**: 오늘 하루 동안 발생한 실제 **실현 손익(매도 확정 수익)**의 누적 합계입니다. (Asset 라인 끝에 표시)

### 전략 관리 및 우선순위 정책
시스템은 각 종목별로 어떤 익절/손절 기준을 따를지 다음의 우선순위에 따라 결정합니다.
1.  **수동 설정 (최우선)**: `3`번 키를 통해 사용자가 직접 수치를 입력한 경우입니다. 화면에 **`[수동]`** 라벨이 표시되며, Vibe 보정이나 프리셋 전략보다 항상 우선합니다.
2.  **프리셋 전략 (차선)**: `9`번 키를 통해 할당된 특정 전략(모멘텀 등)입니다. 해당 **`[전략명]`** 라벨이 표시됩니다.
3.  **시스템 기본 (표준)**: 위 설정이 없는 모든 종목에 적용됩니다. 시장 Vibe 및 시간대별 Phase 보정이 실시간 반영됩니다.

### 트레이딩 로그 및 실적 추적 — `L` 키
*   **TRADE 로그**: 매수/매매 시점, 체결가, 수량 및 **실제 실현 손익**을 기록합니다. 수동 매매 내역도 포함됩니다.
*   **CONFIG 로그**: `S:셋업` 변경, AI 전략 반영, 프리셋 할당 등 시스템의 핵심 설정이 바뀐 이력을 보관합니다.
*   로그 데이터는 `trading_logs.json` 파일에 별도로 저장되어 프로그램 재시작 후에도 유지됩니다.

### 시장 페이즈 (Time-based Phase) 전략
장 시작부터 마감까지 시간대별 변동성을 고려하여 익절/손절선을 추가로 자동 보정합니다.
*   **Phase 1 (09:00~10:00) [OFFENSIVE]**: 장 초반 강한 변동성을 수용하고 수익을 극대화하기 위해 익절선(+2.0%) 상향 및 손절선(-1.0%) 완화가 적용됩니다. (단, 하락장/방어모드 시 손절 완화는 안전을 위해 제외)
*   **Phase 2 (10:00~14:30) [CONVERGENCE]**: 시장 수렴 및 안정화 구간으로, 보수적인 수익 실현을 위해 익절선을 하향(-1.0%) 조정합니다.
*   **Phase 3 (14:30~15:10) [CONCLUSION]**: 장 마감 전 결과 확정 단계로 기준값을 유지합니다.
*   **Phase 4 (15:10~15:20) [PREPARATION]**: 익일 거래를 준비하는 단계입니다.

### AI 로그 (Decision Log) — `A` 키
*   **실시간 투명성 확보**: AI가 백그라운드에서 수행하는 모든 판단 과정을 5개의 탭 UI로 제공합니다. **1초마다 자동 갱신**되어 현재 AI의 상태를 실시간으로 모니터링할 수 있습니다.
*   **[신규] AI 활동 로그 (1번 탭)**: AI 엔진의 주기적 활동(시황분석, 보유분석, MA분석, 매수검토 등)을 시간순으로 기록합니다.
    *   **결과: 0건 (보유분석)**: AI가 보유 종목을 진단했으나, 현재 시점에서 매도나 전략 갱신 등 아무런 조치가 필요하지 않다고 판단(Hold 유지)했음을 의미합니다. 아주 평온한 상태이므로 안심하셔도 됩니다.
    *   **결과 색상**: 성공/승인(`초록`), 실패/거절(`빨강`), 대기/진행(`노랑`)으로 직관적 상태 확인 가능.
*   **매수 거절 로그 (2번 탭)**: 최종 구매 컨펌 단계에서 Gemini가 'No'라고 판단한 종목 리스트와 사유가 표시됩니다. (예: `[G3.1FL]` 단기 과매수 및 테마 소멸 징후 포착)
*   **종목 교체(Replacement) 내역 (3번 탭)**: 포트폴리오 최대 보유 한도(8개) 도달 시, 기존 보유 종목을 매도하고 신규 유망 종목으로 교체 진입한 구체적인 AI 분석 사유가 기록됩니다.
*   **매수 사유 로그 (4번 탭)**: 당일 매수 승인된 종목들의 결정 근거와 사용된 모델 정보를 확인합니다.
*   **전략 수립 로그 (5번 탭)**: 현재 보유 중인 각 종목에 할당된 프리셋 전략의 수립 근거와 AI 분석 의견을 확인합니다.

### 물타기 (하락 방어 전략) — `5` 키
*   보유 종목이 하락할 때 평단가를 낮추기 위해 추가 매수하는 전략입니다.
*   트리거 조건이 충족되면 (자동 모드 ON 시) 시스템이 자동으로 집행합니다.
*   **안전장치 1**: 물타기 트리거는 항상 손절선(SL)보다 높게 유지됩니다.
*   **안전장치 2 (핑퐁 방지)**: 물타기 집행 직후 **30분간 즉각 손절 유예**를 적용합니다. 단, 아래 긴급 조건 발생 시 즉시 손절합니다:
    *   손실률이 SL보다 -1.0% 추가 하락 | 글로벌 패닉 | 방어모드(Defensive) 전환 | 장마감(P4)+손실권
*   **안전장치 3 (재진입 차단)**: 손절 발생 후 **2시간** 동안 자동 물타기 재진입에 쿨다운을 적용합니다 (연속 핑퐁 소진 방지).

### 불타기 (수익 확대 전략) — `6` 키
*   보유 종목이 상승 추세일 때 수익 비중을 확대하기 위해 추가 매수하는 전략입니다.
*   상승장(Bull) 또는 거래량 폭발 시에만 작동하며, 직전 매수가 대비 +2.0% 이상 추가 상승했을 때 집행합니다.
*   **안전장치 1**: 불타기 트리거는 항상 익절선(TP)보다 -1.0% 이상 낮게 제한되어, 매수 직후 익절에 걸리는 핑퐁 매매를 방지합니다.
*   **안전장치 2 (쿨다운 리셋)**: 불타기 집행 시 익절 쿨다운이 자동 리셋됩니다 → 새 포지션에서 TP 도달 즉시 익절이 허용됩니다.
*   **안전장치 3 (긴급 익절 바이패스)**: 쿨다운 중이라도 아래 조건 충족 시 즉시 익절합니다:
    *   수익률 ≥ TP + **2.0%** 초과 (불타기 직후) / 일반 쿨다운 시 **+3.0%**
    *   거래량 폭발(Vol Spike) + 수익률 ≥ TP + **1.0%** (불타기 직후) / 일반 +1.5%
    *   **Phase 4 (장 마감)** + 수익권
*   **안전장치 4 (재진입 차단)**: 익절 발생 후 **2시간** 동안 자동 불타기 재진입에 쿨다운을 적용합니다 (연속 핑퐁 소진 방지).

### 텔레그램 알림 엔진 (Notification) — [v1.6.0 신규]
언제 어디서나 스마트폰으로 트레이딩 현황을 모니터링할 수 있는 텔레그램 알림 기능을 지원합니다.

*   **주요 알림 내역**:
    *   **실시간 매매**: 매수/매도/익절/손절 발생 시 즉시 상세 내역(수익금, AI 사유 포함) 발송.
    *   **정기 리포트**:
        *   `09:00`: 장 개시 알림 및 오늘의 시장 VIBE 요약.
        *   `15:30`: 장 마감 알림 및 당일 최종 손익/자산 현황 요약.
        *   `16:00`: 투자 복기 리포트(Retrospective) 완료 시 핵심 요약 발송.
    *   **긴급 경보**: 시장 VIBE 급변(Bull↔Bear), 서킷 브레이커(Halted) 발동/해제 시 즉시 알림.
*   **설정 방법**:
    1.  텔레그램에서 [@BotFather](https://t.me/botfather)를 통해 봇을 생성하고 **토큰(Token)**을 발급받습니다.
    2.  [@myidbot](https://t.me/myidbot)에게 `/getid`를 보내 자신의 **Chat ID(숫자)**를 확인합니다.
    3.  **반드시** 새로 만든 봇의 대화창에 들어가서 **[시작]** 또는 `/start`를 눌러야 알림 수신이 가능합니다.
    4.  프로그램의 `S:셋업` 메뉴에서 토큰과 ID를 입력하면 즉시 가동됩니다.

*   **AI 시황 자동 반영**: `S:셋업`에서 이 기능을 활성화하면, `8:시황` 분석 후 별도의 컨펌 없이 AI가 도출한 최적 TP/SL 수치가 시스템에 즉시 반영됩니다.

### 텔레그램 원격 제어 (Inbound Commands) — [신규]
텔레그램 봇 채팅방에 명령어를 입력하여 스마트폰 등 외부에서 시스템을 실시간으로 제어할 수 있습니다. 시스템 구동 중에만 동작합니다.
*   **보안**: `S:셋업`에 입력된 `TELEGRAM_CHAT_ID`와 일치하는 사용자만 명령을 실행할 수 있습니다.
*   **명령어 목록**:
    *   `/status` : 📊 현재 총 자산, 예수금, 당일 수익금 및 VIBE/패닉 상태 요약 확인
    *   `/buy [종목코드] [수량] [가격]` : 📈 수동 매수 (가격 미입력 시 시장가 주문). 예) `/buy 005930 10`
    *   `/sell [종목코드] [수량] [가격]` : 📉 수동 매도 (가격 미입력 시 시장가 주문). 예) `/sell 005930 10 80000`
    *   `/panic` : 🚨 **긴급 패닉 발동**. 신규 매수를 즉시 차단하고 보유 중인 전 종목을 즉시 긴급 청산(시장가 매도) 시도
    *   `/pause` : ⏸️ **매수 일시정지**. 보유 종목 익절/손절 관리는 유지하되, 신규 매수만 차단
    *   `/resume`: ▶️ **매매 재개**. 일시정지 상태를 해제
    *   `/defensive` : 🛡️ **방어모드 강제 전환**. AI 시황과 무관하게 시스템을 극보수적 방어모드로 강제 변경
    *   `/reset` : 🔄 **전면 정상화**. 패닉, 일시정지, 강제 방어모드 등 외부에서 개입한 모든 특수 상태를 한 번에 해제하고 다시 AI 자율 운용 체제로 복귀

### AI 시황 분석 (`8`) 전략 자동 적용

#### AI 동적 수치 계산 로직 (Dynamic TP/SL)
*   **원리**: 사용자가 전략을 선택하면, Gemini AI가 해당 종목의 **[현재가 흐름 + 재무 지표 + 최근 뉴스 + 시장 장세]**를 입체적으로 분석합니다.
*   **미세 조정**: 선택한 전략의 기본값(예: 이격도 +5%)을 베이스로, AI가 판단한 종목의 실시간 변동성에 따라 수치를 **±2~4% 범위 내에서 자동 조정**합니다.
*   **결과**: 동일한 종목에 다른 전략을 적용하더라도, AI가 분석한 해당 종목의 적정 리스크(체력)가 동일하다면 TP/SL 수치가 같게 나올 수 있습니다. 이는 시스템 오류가 아닌 **종목 맞춤형 최적화 결과**입니다.

*   **우선순위**: 프리셋 전략이 할당된 종목은 기미 설정된 글로벌 TP/SL보다 **프리셋이 최우선으로 적용**됩니다.
*   **자동 매수 연동**: AI 자율 매수 모드(AUTO)에서 신규 매수 성공 시 프리셋 전략이 자동으로 할당됩니다.
*   **영속성**: 할당된 전략은 `trading_state.json`에 저장되어 프로그램 재시작 시에도 유지됩니다.
*   **TUI 표시**: 보유 종목 리스트의 TP/SL 우측에 `[골든크로스]` 등의 태그가 시안색으로 표시됩니다.
### 성과 대시보드 및 리밸런싱 (Hall of Fame) — `P` 키
*   **명예의 전당 (수익/손실 TOP 10)**: 지금까지 매매를 통해 가장 큰 누적 수익 및 손실을 안겨준 **'명예의 전당(Hall of Fame)'** 및 **'불명예의 전당(Hall of Shame)'** 종목 리스트를 가시화합니다.
*   **모델별 통합 성과**: 결정 주체(G3.1 Pro, G3.1 Flash, 시스템 TP/SL, 수동 등)별로 승률(Win Rate)과 누적 수익금을 상세히 집계하여, 어떤 결정 로직이 포트폴리오에 기여했는지 데이터로 증명합니다.
*   **AI 리밸런싱 제안**: 
    *   Gemini가 현재 포트폴리오의 **비중(Weight)**과 **수익률**을 분석하여 핵심 리밸런싱 의견을 제시합니다.
    *   특정 종목의 비중이 과도(30% 이상)하거나 성과가 정체된 경우, 비중 축소 또는 유망 종목으로의 교체를 권장합니다.
    *   `R` 키를 눌러 실시간으로 리밸런싱 분석을 갱신할 수 있습니다.

## 5. 문제 해결 (FAQ)

*   **Q: 데이터가 갱신되지 않아요.**
    *   A: 인터넷 연결을 확인하거나, KIS API 키가 만료되지 않았는지 확인하세요. `S` 키를 눌러 설정을 재검토할 수 있습니다.
*   **Q: AI 분석(7/8)이 동작하지 않아요.**
    *   A: `GOOGLE_API_KEY`가 올바르게 입력되었는지 확인하세요. 무료 버전의 경우 초당 호출 제한이 있을 수 있습니다. AI 장애 시 기존 알고리즘 모드로 자동 전환됩니다.
*   **Q: EXE 파일 실행 시 경고 창이 떠요.**
    *   A: 서명되지 않은 실행 파일의 경우 윈도우에서 경고가 뜰 수 있습니다. [추가 정보] 클릭 후 [실행]을 누르세요.
*   **Q: 화면이 깨져 보여요.**
    *   A: 터미널 창 크기를 조절하면 자동으로 다시 그려집니다. 전체 화면 모드(리포트/분석) 후 ESC로 복귀 시에도 자동 갱신됩니다.
