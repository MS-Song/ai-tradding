# Phase 4: 외부 연결성 및 사용자 인터페이스 확장 (Connectivity)

## 1. 개요 (Goal)
터미널 UI의 한계를 넘어 실시간 모바일 알림과 웹 기반의 대시보드를 제공하여 사용자가 언제 어디서나 시스템 상태를 명확히 모니터링할 수 있게 합니다.

## 2. 상세 구현 항목

### A. 실시간 모바일 알림 봇 (Telegram Messenger)
- **기능**: 주요 체결 내역, 급격한 시장 변화, AI 긴급 공지 등을 텔레그램으로 즉시 알립니다.
- **구축 내용**:
    - `src/utils/notifier.py` 구현.
    - `requests`를 이용한 Telegram Bot API 연동.
    - 매수/매도 체결 직후 비동기로 메시지 전송.
- **기대 효과**: PC를 보고 있지 않아도 트레이딩 상황을 실시간 파악 가능.

### B. 웹 대시보드 및 API 서버 (Monitoring Dashboard)
- **기능**: `trading_logs.json` 및 `trading_state.json` 데이터를 시각화하여 웹 브라우저에서 볼 수 있도록 합니다.
- **구축 내용**:
    - `FastAPI` 또는 `Flask` 기반의 초경량 웹 서버 도입. (종료 시 동시 종료되도록 `sub-process`로 관리)
    - 자산 추이 그래프 (Chart.js 활용), 실시간 수익률 히트맵 제공.
    - `docs/dashboard.html` (Single Page App) 형태의 정적 파일 제공.
- **기대 효과**: 텍스트 위주의 정보를 직관적인 그래프로 전환하여 자산 흐름 파악 용이.

### C. 원격 제어 인터페이스 (Basic Commands)
- **기능**: 외부(웹 또는 텔레그램)에서 간단한 명령(전체 매도, 봇 일시정지)을 수행할 수 있게 합니다.
- **구축 내용**:
    - 텔레그램 명령 처리기(Command Handler) 구현.
    - 웹 API를 통한 `trading_state.json` 상의 플래그 제어.
- **기대 효과**: 긴급 상황 발생 시 원격 대응 능력 확보.

## 3. 검증 계획
- **전송 지연 테스트**: 사건 발생 후 텔레그램 도착까지의 지연 시간(평균 2초 이내) 확인.
- **웹 호환성**: 크롬/사파리 모바일 브라우저에서 대시보드 레이아웃의 정상 작동 및 데이터 실시간 업데이트 확인.
