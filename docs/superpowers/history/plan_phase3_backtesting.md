# Phase 3: 백테스팅 엔진 및 전략 저장소 (Validation & Archive)

## 1. 개요 (Goal)
과거 데이터를 기반으로 현재의 AI 전략을 시뮬레이션하여 리스크와 기대 수익을 사전에 검증하고, 검증된 전략을 영속적으로 저장하고 재사용할 수 있는 체계를 구축합니다.

## 2. 상세 구현 항목

### A. Historical Data Downloader & DB
- **기능**: 백테스팅에 필요한 과거 일봉/분봉 데이터를 KIS API를 통해 대량으로 내려받아 로컬에 저장합니다.
- **구축 내용**:
    - `scripts/data_collector.py` 구현.
    - SQLite3 또는 로컬 JSON 파일을 활용한 대용량 데이터 저장소 구축.
    - 최근 1년치 일봉 및 최근 1개월치 분봉 확보 로직.
- **기대 효과**: API 호출 제한에 걸리지 않고 반복적인 전략 검증 수행 가능.

### B. 가상 트레이딩 시뮬레이터 (Simulator Core)
- **기능**: 실제 KIS API와 동일한 인터페이스를 가지되, 로컬 데이터를 바탕으로 작동하는 `MockAPI`를 구현하여 `VibeStrategy`를 돌립니다.
- **구축 내용**:
    - `tests/simulator.py` 신설.
    - 가상 시간(Virtual Clock) 제어를 통해 1년치 매매를 수분 내에 완료하는 구조.
    - 슬리피지(Slippage) 및 수수료(0.015%) 설정 반영.
- **기대 효과**: 실제 자산을 투입하기 전 전략의 Max Drawdown(MDD)과 승률 파악.

### C. 전략 저장소 (Strategy Repository)
- **기능**: 특정 시장 상황에서 우수했던 AI 전략 설정(TP/SL, AI 분석 의견 등)을 저장하고 유사한 장세가 오면 자동으로 재로드합니다.
- **구축 내용**:
    - `src/strategy/repository.py` 신설.
    - 수익률 기준 Top 10 전략 아카이빙.
    - 장세(Vibe) 태그를 기반으로 한 전략 검색 및 추천 기능.
- **기대 효과**: 매번 처음부터 분석하는 대신, '성공 경험'을 축적하는 유기적 엔진으로 진화.

## 3. 검증 계획
- **정합성 테스트**: 동일 기간에 대해 실전 매매 결과와 백테스팅 결과의 오차가 5% 이내인지 확인.
- **보고서 생성**: 백테스팅 완료 후 샤프 지수(Sharpe Ratio) 및 MDD가 포함된 Markdown 리포트 자동 생성 검증.
