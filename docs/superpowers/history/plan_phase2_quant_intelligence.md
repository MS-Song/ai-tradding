# Phase 2: 기술적 분석 엔진 및 AI 지능 고도화 (Quant Intelligence)

## 1. 개요 (Goal)
Gemini AI에게 뉴스나 펀더멘털뿐만 아니라, 정교한 기술적 보조지표 데이터를 제공하여 매매 타이밍의 정확도를 획기적으로 향상시킵니다.

## 2. 상세 구현 항목

### A. QuantIndicator 전용 모듈 구축
- **기능**: RSI, 볼린저 밴드, MACD 등 핵심 기술적 지표를 실시간으로 계산하여 제공합니다.
- **구축 내용**:
    - `src/strategy/indicator_engine.py` 신설.
    - KIS API의 분봉(1분, 10분) 차트 데이터를 활용하여 지표 계산.
    - `pandas` 라이브러리 도입 검토 (데이터 처리의 효율성을 위해).
- **기대 효과**: 객관적인 과매수/과매도 수치를 근거로 한 냉철한 판단 가능.

### B. AI 프롬프트 엔지니어링 고도화 (3D-Insight)
- **기능**: 이전의 '뉴스+지수' 데이터에 '기술적 지표'를 결합한 3D 입체 분석 프롬프트로 강화합니다.
- **구축 내용**:
    - `Advisor.provide_insight` 메서드 업데이트.
    - 프롬프트에 "RSI: 75 (과매수 주의)", "볼린저 상단 돌파" 등의 문구 추가.
    - AI가 수치 데이터를 기반으로 TP/SL의 정밀도를 ±0.5% 단위로 조정하도록 유도.
- **기대 효과**: 감성 분석(뉴스)과 계량 분석(지표)의 상호 보완.

### C. 멀티 모델 A/B 성능 트래킹 (Model Performance Ledger)
- **기능**: 여러 Gemini 모델 중 어떤 모델이 특정 전략에서 가장 우수한 수익률을 내는지 기록하고 비교합니다.
- **구축 내용**:
    - `trading_logs.json`에 `model_id` 필드 강화.
    - 매도 시점에 해당 종목의 전략을 수립한 모델 정보를 매칭하여 수익률 합산.
    - `A: AI로그 리포트`에 모델별 당일 수익률 순위표 추가.
- **기대 효과**: 최적의 모델-전략 조합(Model-Strategy Fit) 발견.

## 3. 검증 계획
- **지표 검증**: HTS/MTS의 지표 수치와 시스템 계산 수치의 오차 범위(±0.1%) 확인.
- **프롬프트 테스트**: 시뮬레이션 환경에서 지표 변화에 따른 AI 가이던스의 변화 양상 모니터링.
