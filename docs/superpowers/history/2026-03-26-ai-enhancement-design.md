# 📄 Spec: AI Trading Engine Enhancement (2026-03-26)

## 1. 개요
본 문서는 추천 종목 시스템 최적화, 과거 추천 성과 트래킹, 그리고 개별 종목 심층 분석(AI 분석) 기능 강화를 위한 상세 설계 내용을 담고 있습니다. KIS API 호출을 최소화하고 외부 데이터(Naver/Yahoo)를 적극 활용하여 시스템 안정성을 확보하는 것이 핵심입니다.

## 2. 주요 기능 상세 설계

### A. 추천 종목 시스템 (5+1 구조)
*   **데이터 필터링**: `VibeAlphaEngine.analyze` 결과에서 일반 종목 상위 5개와 ETF 상위 1개를 선정.
*   **정렬 기준**: AI 점수(Score) 내림차순.
*   **UI 배치 (TUI)**: 
    *   Row 1: 일반 종목 1, 2, 3위
    *   Row 2: 일반 종목 4, 5위 + ETF 1위
*   **데이터 소스**: 네이버 금융 실시간 시세 (KIS API 미호출).

### B. 어제 추천 종목 트래킹 (Yesterday's Recs)
*   **영속성**: `trading_state.json`의 `last_recs` 필드에 `[{code, price, date}]` 형태로 저장.
*   **갱신 로직**: 앱 실행 시 저장된 날짜가 오늘 이전인 경우, 해당 데이터를 '어제 추천' 세션으로 로드.
*   **히트율 분석**: 
    *   네이버 시세를 통해 현재가 획득.
    *   변동률 = `(현재가 - 추천가) / 추천가 * 100`.
    *   절대값 변동률이 큰 상위 3개 종목을 TUI 랭킹 하단에 노출.

### C. 메뉴 개편 및 AI 종목 분석 (7번/8번)
*   **메뉴 단축키 변경**:
    *   `7`: **AI 종목분석** (신규) - 특정 종목에 대한 뉴스/공시 기반 심층 리포트.
    *   `8`: **AI 시황분석** (기존 'A' 기능 명칭 변경).
    *   `9`: **필터** (메뉴판에서만 제거, 키 입력 시 로직은 그대로 유지).
*   **AI 종목분석(7) 프로세스**:
    1.  사용자로부터 종목 코드(6자리) 입력받음.
    2.  네이버 금융 뉴스 및 공시 탭 헤드라인 크롤링.
    3.  네이버 금융 종목 상세 지표(PER, PBR 등) 수집.
    4.  Gemini AI 프롬프팅: "이 가격 변동의 원인은 무엇인가?", "모멘텀이 내일까지 유지될 것인가?", "매수/매도 고민 해결사" 관점의 리포트 생성.
    5.  `D: 리포트`와 유사한 전체 화면 UI로 출력.

## 3. 문서화 및 관리 정책
*   **설계서 업데이트**: 주요 기능 변경 시 `gemini.md` 내 설계 사상을 즉시 반영함.
*   **매뉴얼 동기화**: 모든 사용자 인터페이스(UI) 및 단축키 변경 사항은 `docs/USER_MANUAL.md`에 즉시 업데이트함.

## 4. 기술적 제약 사항
*   데이터 수집은 KIS API 부하 방지를 위해 Naver Finance와 Yahoo Finance API/크롤러를 우선 사용함.
*   KIS API는 오직 실시간 잔고 확인 및 실제 주문 체결 업무에만 할당함.
