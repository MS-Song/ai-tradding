# Phase 1: 리스크 관리 및 시스템 회복력 강화 (Risk & Resilience)

## 1. 개요 (Goal)
개별 종목의 변동성을 반영한 지능형 매수 수량 조절(Position Sizing)을 도입하고, 시스템 전체의 손실을 방어하는 서킷 브레이커와 API 계층의 안정성을 강화합니다.

## 2. 상세 구현 항목

### A. ATR 기반 지능형 포지션 사이징
- **기능**: 단순히 고정 금액(예: 50만원)을 매수하는 것이 아니라, 종목의 최근 변동성(ATR)을 측정하여 수량을 자동 조절합니다.
- **구축 내용**:
    - `src/strategy/risk_manager.py` 신설.
    - KIS API의 일봉 차트 서비스를 호출하여 최근 14일간의 ATR(Average True Range) 계산.
    - 매수 수량 = `(총 자산 * 리스크 단위) / (ATR * 배수)`.
- **기대 효과**: 급등락이 심한 종목은 적은 수량을, 안정적인 종목은 더 많은 수량을 매수하여 포트폴리오의 리스크 평준화.

### B. 포트폴리오 서킷 브레이커 (Circuit Breaker)
- **기능**: 당일 누적 평가 손실이 설정된 임계치(예: 총 자산의 3%)를 초과할 경우, 모든 신규 매수를 즉시 중단하고 모니터링 모드로 전환합니다.
- **구축 내용**:
    - `DataManager`가 자산 상태를 추적할 때 `daily_pnl_rate`를 계산.
    - `VibeStrategy.run_cycle` 진입 시 `RiskManager.check_barrier()`를 호출하여 차단 여부 판단.
- **기대 효과**: 블랙 스완 발생 시 비이성적인 물타기나 추가 매수 방지.

### C. API Resilience & Auto-Recovery
- **기능**: KIS API 호출 시 발생하는 일시적 장애나 속도 제한(429)을 자동으로 처리합니다.
- **구축 내용**:
    - `src/utils.py`에 `@retry_api` 데코레이터 구현.
    - 지수 백오프(Exponential Backoff) 및 최대 3회 재시도.
    - 토큰 만료 에러 감지 시 즉각 `auth.generate_token()` 호출 후 재시도.
- **기대 효과**: 네트워크 불안정 환경에서도 중단 없는 트레이딩 수행.

## 3. 검증 계획
- **단위 테스트**: `tests/test_risk_manager.py`를 통해 ATR 계산 로직 검증.
- **모의 환경**: 모의 매매 계좌에서 인위적으로 손실 상황을 연출하여 서킷 브레이커 작동 확인.
