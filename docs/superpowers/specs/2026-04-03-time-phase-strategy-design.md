# 📄 Spec: AI-Linked Time-Phase Strategy (2026-04-03)

## 1. 개요
본 문서는 KIS-Vibe-Trader 트레이딩 엔진에 '시간(Time)' 개념을 도입하여 자금 회전율을 극대화하고 단기 차익 실현을 자동화하기 위한 상세 설계를 기술함.

## 2. 핵심 메커니즘

### A. 장중 3단계 시간 페이즈 (Market Phase)
현재 시각에 따라 시스템 전체의 TP/SL 기준을 동적으로 보정함.

| 페이즈 | 시간대 | 명칭 | 전략적 가중치 | 예외 조건 |
| :--- | :--- | :---: | :--- | :--- |
| **Phase 1** | 09:00~10:00 | **공격** | TP +2.0% / SL -1.0% | Vibe가 Bear/Defensive일 경우 SL 완화 제외 |
| **Phase 2** | 10:00~14:30 | **수렴** | TP -1.0% / SL -1.0% | 없음 |
| **Phase 3** | 14:30~15:20 | **확정** | 수익권(+0.5%↑) 50% 시장가 매도 | 잔량 SL을 본전으로 강제 상향 |

### B. 전략별 AI 동적 데드라인 (Time-Stop)
전략 할당 시 AI가 종목의 모멘텀 수명을 예측하여 매도 데드라인을 설정함.

1.  **AI 프롬프트 확장**: `simulate_preset_strategy` 호출 시 `유효시간(분)` 응답 추가.
2.  **그룹별 안전 상한선 (Hard Cap)**:
    *   G1 (돌파형): 최대 180분
    *   G2 (추세형): 당일 마감
    *   G3 (반등형): 최대 240분
3.  **데드라인 초과 액션**: 지정된 시각 경과 시 익절선(TP)을 현재 수익률의 50% 수준(최소 +0.5%)으로 하향하여 즉시 수익 확정 유도.

## 3. 데이터 구조 변경 (`trading_state.json`)
`preset_strategies` 내부의 각 종목 객체에 아래 필드 추가:
*   `buy_time`: 매수 시각 (YYYY-MM-DD HH:MM:SS)
*   `deadline`: 마감 예정 시각 (YYYY-MM-DD HH:MM:SS)
*   `is_p3_processed`: Phase 3(절반 매도) 집행 여부 (Boolean)

## 4. UI/UX 변경 사항
*   **보유 종목 리스트**: 'STGY' 컬럼 우측에 **[남은 시간]** 또는 **[D-Line 시각]** 표시.
*   **상태바**: 현재 어떤 Phase(공격/수렴/확정)인지 모드 아이콘 표시.

## 5. 기술적 제약 및 안전 장치
*   **Vibe 우선 원칙**: 시간 페이즈의 완화 정책보다 시장 전체의 Vibe(Panic/Defensive) 리스크 관리 정책이 우선함.
*   **중복 실행 방지**: Phase 3의 50% 매도는 당일 종목당 1회만 실행되도록 `is_p3_processed` 플래그로 제어함.
