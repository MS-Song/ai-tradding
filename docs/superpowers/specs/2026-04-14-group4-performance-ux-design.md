# 📑 Spec: Group 4 - Multi-threading Performance & UX

## 1. 개요
TUI의 반응 속도를 개선하기 위해 입력과 실행 스레드를 분리하고, 주요 지표에 색상을 적용하여 시인성을 극대화합니다.

## 2. 주요 기능
### A. 비동기 커맨드 실행 (요구사항 4)
- **입력 스레드 분리**: `interaction.py`의 키 입력을 처리하는 동안 렌더링 스레드가 멈추지 않도록 구조 변경.
- **작업 큐(Task Queue) 도입**: 사용자가 내린 명령(매수, 매도, 리포트 생성 등)을 큐에 넣고 별도의 Worker 스레드에서 실행.
- **실행 중 피드백**: 명령이 수행되는 동안에도 화면은 갱신되며, `[처리 중...]` 등의 상태 표시기 노출.

### B. UI 시인성 강화 (요구사항 9-색상/레이아웃)
- **색상 적용 (ANSI Color)**:
    - 익절 수치 (+): **빨간색** / 손절 수치 (-): **파란색**.
    - 자동 모드 ON: **노란색** / OFF: **회색**.
- **레이아웃 정렬**:
    - `Asset` 라인 파이프(`|`) 위치 및 대문자 표기 통일.
    - `BEAR | BULL | ALGO` 라인 끝에 **금일 누적 발생 금액** 표기.
    - 기존 설명 문구("PnL 하락 방어" 등) 제거하여 여백 확보.

## 3. 구현 세부 사항
- `src/ui/interaction.py` 내의 `perform_interaction`을 `threading.Thread`로 래핑하여 호출.
- `src/ui/renderer.py`의 `draw_tui` 내에서 수익률에 따른 색상 코드(`\033[...]`) 주입 로직 추가.
