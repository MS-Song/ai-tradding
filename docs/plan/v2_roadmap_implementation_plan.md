# 🚀 KIS-Vibe-Trader V2.0 로드맵 구현 계획서

본 계획서는 시니어 아키텍트의 설계 사상을 반영하여, KIS-Vibe-Trader를 단순 자율 매매 엔진에서 **'기관급 리스크 관리 및 멀티 모델 지능을 갖춘 통합 트레이딩 플랫폼'**으로 격상시키기 위한 구체적인 기술 로드맵을 담고 있습니다.

---

## 1. 개요 (Overview)
- **목표**: 실시간 모니터링 강화, 리스크 분산 고도화, 의사결정 신뢰도 향상.
- **핵심 가치**: [지키는 투자]의 기술적 완성도 극대화 및 사용자 경험(UX) 혁신.
- **주요 모듈**: `Notifier`, `PortfolioGuard`, `EnsembleAdvisor`, `VolatilityEngine`.

---

## 2. 상세 구현 계획 (Phase-by-Phase)

### Phase 1: 외부 연결성 및 기초 자산 보호 (Connectivity & Basic Safety)
**목표**: 시스템의 생존 여부를 실시간으로 파악하고, 유휴 현금의 기회비용을 최소화함.

#### 1.1 알림 엔진 (`Notifier`) 구현
- **기술 스택**: `python-telegram-bot` 또는 웹훅(Webhook) 기반 디스코드 연동.
- **구현 상세**:
    - `src/utils/notifier.py` 신규 생성.
    - **실시간 매매 알림**: `log_trade` 발생 시 즉시 텔레그램 메시지 발송 (종목명, 수량, 가격, AI 사유 포함).
    - **정기 리포트**: 09:00(장 개시), 15:30(장 마감 후), 16:00(복기 리포트) 자동 전송.
    - **임계치 경보**: 서킷 브레이커 발동, VIBE 급변(Bull->Bear) 시 즉시 경보.

#### 1.2 스마트 현금 관리 (`Smart Cash`) 로직 추가
- **구현 상세**:
    - `RecoveryEngine` 및 `PyramidingEngine` 내 현금 비중 체크 로직 고도화.
    - `Defensive` 모드 진입 시, 가용 현금의 90%를 미리 지정된 '파킹형 ETF'(예: 403870, 465520)로 자동 매수.
    - 신규 매수 신호 발생 시, 파킹형 ETF를 우선 매도하여 현금을 확보하는 `LiquidityManager` 기능 구현.

---

### Phase 2: 지능형 리스크 분산 및 앙상블 (Intelligence & Risk)
**목표**: 특정 섹터 쏠림을 방지하고 AI의 판단 신뢰도를 극대화함.

#### 2.1 섹터 및 상관관계 분석기 (`PortfolioGuard`)
- **구현 상세**:
    - 네이버 금융의 업종 정보를 활용하여 보유 종목의 섹터 맵핑 자동화.
    - **섹터 캡(Sector Cap)**: 특정 업종 비중이 30% 초과 시 AI 추천 점수에 관계없이 해당 업종 신규 매수 차단.
    - **상관관계 계산**: 최근 20일 종가 데이터를 기반으로 종목 간 상관계수(Correlation) 산출.
    - 상관계수 0.8 이상의 종목이 포트폴리오에 있을 경우, 유사 종목 진입을 지양하도록 AI에게 페르소나 주입.

#### 2.2 멀티 모델 앙상블 (`EnsembleAdvisor`)
- **구현 상세**:
    - `Advisors` 폴더 내에 `EnsembleManager` 추가.
    - **Voting System**: `confirm_buy_decision` 호출 시 Gemini 3.1 Pro와 Llama 3.1 70B(Groq)에 동시 질의.
    - **합의 알고리즘**: 두 모델 모두 'Yes'일 때만 진입하는 'Strict Mode'와 한 모델이라도 강력 추천 시 진입하는 'Flexible Mode' 선택 기능.
    - 각 모델의 분석 로그를 `A:로그` 화면에 병렬로 가시화.

---

### Phase 3: 동적 전략 보정 및 시스템 최적화 (Dynamic Strategy & DX)
**목표**: 시장 변동성에 따라 전략을 유연하게 수정하고 시스템의 가용성을 보장함.

#### 3.1 변동성 적응형 엔진 (`VolatilityEngine`)
- **구현 상세**:
    - `MarketAnalyzer`에 시장 변동성 지표(VIX 또는 코스피 20일 표준편차) 통합.
    - **동적 TP/SL**: 고변동성 장세에서는 `TP +3%, SL -2%` 식으로 폭을 넓히고, 저변동성에서는 폭을 좁혀 수익을 확정하는 로직.
    - ATR(Average True Range) 기반의 익절 트레일링 스탑 정교화.

#### 3.2 시스템 워치독 및 DX 개선
- **구현 상세**:
    - **Watchdog 스레드**: 각 Worker Thread의 생존 여부를 1분 주기로 체크하여 좀비 스레드 발생 시 자동 재생성.
    - **TUI 로그 검색**: 단축키 `/`를 통한 실시간 로그 검색 및 종목별 로그 필터링 기능.
    - **설정 자동 튜닝**: `RetrospectiveEngine`의 복기 데이터를 기반으로 승률이 높은 시간대/종목군의 가중치를 자동으로 상향 조정하는 피드백 루프.

## 3. 검증 계획 (Verification Plan)

### 3.1 단위 테스트 (Unit Testing)
- `Notifier`: 가상 메시지 전송 테스트 및 네트워크 장애 시 큐잉(Queuing) 동작 확인.
- `PortfolioGuard`: 특정 섹터 종목 8개 주입 시 9번째 종목 매수 차단 여부 검증.

### 3.2 통합 시뮬레이션 (Integration Test)
- **모의투자 모드**: 1주일간 멀티 모델 앙상블 모드로 운영하며 단일 모델 대비 승률 및 MDD(최대 낙폭) 비교.
- **Fail-over 테스트**: Gemini API 키를 강제로 만료시킨 후 Groq로 1초 이내 전환되는지 확인.
